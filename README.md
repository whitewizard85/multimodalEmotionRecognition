# Multimodal Emotion Recognition — Audio, Text, Visual Fusion

Companion code for a multimodal emotion recognition system that fuses a
fine-tuned wav2vec 2.0 speech model, a fine-tuned BERT/RoBERTa text model,
and a fine-tuned ResNet50 facial-expression model via decision-level
probability averaging.

## Honest framing, up front

This repository is not the code behind a paper's original claimed numbers —
it's the code that **replaced** them after a re-verification turned up a
train/test leakage bug (augmentation applied before, not after, the
train/test split) that had inflated the audio and text branches' reported
accuracy, and an architecture mismatch in the visual branch (a small CNN
trained from scratch, despite being labeled "ResNet-50" in the paper's own
results table). Both issues are fixed here. The honestly-evaluated results
are lower than what was originally reported, and the repository includes
the diagnostic scripts (`test_leaky_hypothesis.py`,
`test_leaky_visual_hypothesis.py`) that were used to *deliberately
reproduce* the leakage bug and quantify its effect (a 28-point accuracy
inflation on the audio branch alone), rather than just asserting the fix
mattered.

If you're evaluating whether to trust or reuse this code, that history is
probably the single most useful thing to know about it.

## Results

Evaluated on held-out test/validation splits, with the train/test split
always performed *before* any data augmentation:

| Modality | Accuracy | Macro F1 |
|---|---|---|
| Audio (wav2vec2, fine-tuned) | 84.0% | 0.853 |
| Text (BERT, fine-tuned) | 74.4% | 0.685 |
| Visual (ResNet50, fine-tuned) | 55.7% | 0.546 |
| **Full fusion (Audio+Text+Visual)** | **93.5% ± 0.41** | **0.924 ± 0.006** |

Fusion and ablation numbers are the mean of 20 independent random pairings
of the synthetic evaluation set (see **Evaluation protocol and caveats**
below) — not a single run. See `results/ablation.csv` (generated by
`evaluate_fusion.py`) for the full modality-subset ablation and the
missing-modality robustness test.

## Evaluation protocol and caveats

**This is the most important section to read before trusting these
numbers.** Two things to know:

1. **Fusion is evaluated on synthetically paired samples.** The audio,
   text, and visual test sets are independently collected — there is no
   dataset here with simultaneous audio+text+face recordings of the same
   expressive instance. `evaluate_fusion.py` pairs samples across
   modalities by matching emotion label only. Because paired samples come
   from *unrelated* underlying instances, whatever makes one modality's
   sample hard to classify has no relationship to what makes another
   modality's sample hard — unlike real multimodal data, where a genuinely
   ambiguous moment tends to be ambiguous across voice, face, and words
   *together*. This likely makes fusion easier (and the reported gain
   larger) than it would be on real paired data. The 20-seed variance check
   shows the numbers are *reproducible* under this procedure; it does not
   show the procedure is *representative* of real multimodal fusion.

2. **The visual branch's accuracy (55.7%) is modest and likely reflects
   dataset heterogeneity more than backbone capacity** — the four
   constituent datasets (RAF-DB, AffectNet, ExpW, FER2013) differ
   substantially in image quality, pose, and labeling convention, and
   FER2013 specifically is documented in the literature as containing
   label noise.

## Pipeline overview

```
Raw audio (RAVDESS/CREMA-D/TESS/SAVEE)
    -> train_audio_wav2vec2.py  -> audio_wav2vec2_model/
    -> predict_audio.py         -> audio_probs.npz

Raw text (GoEmotions/dair-ai/emotion)
    -> train_text.py (or train_text_roberta.py) -> text_model/
    -> predict_text.py          -> text_probs.npz

Raw face images (RAF-DB/AffectNet/ExpW/FER2013)
    -> build_visual_dataset.py  -> visual_dataset/{train,val}/<emotion>/
    -> train_visual_resnet50.py -> visual_resnet50_model.h5
    -> predict_visual.py        -> visual_probs.npz

audio_probs.npz + text_probs.npz + visual_probs.npz
    -> evaluate_fusion.py       -> results/table5_corrected.csv, results/ablation.csv

results/table5_corrected.csv
    -> generate_figures.py      -> confusion matrices + results bar chart
```

## Two Python environments are required

TensorFlow and PyTorch fighting over CUDA/cuDNN versions in the same
virtual environment caused repeated, hard-to-diagnose breakage during
development. The scripts are split accordingly:

- **`venv` (PyTorch + Transformers)**: `train_audio_wav2vec2.py`,
  `train_text.py`, `train_text_roberta.py`, `predict_audio.py`,
  `predict_text.py`, `train_visual_facenet.py`, `align_faces_mtcnn.py`
- **`venv_visual` (TensorFlow only)**: `train_visual.py`,
  `train_visual_resnet50.py`, `predict_visual.py`, `build_visual_dataset.py`
- **Either environment** (numpy/pandas/sklearn/matplotlib only):
  `evaluate_fusion.py`, `generate_figures.py`, `common.py`,
  `audio_label_utils.py`

```bash
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
pip install torch==2.4.1 torchvision==0.19.1 --index-url https://download.pytorch.org/whl/cu121

# separately, for the TensorFlow-only environment:
python3 -m venv venv_visual && source venv_visual/bin/activate
pip install "tensorflow[and-cuda]==2.17.0" scikit-learn numpy pillow
```

See `requirements.txt` for the full pinned versions and notes on why each
pin exists (several were the direct result of debugging real CUDA/version
conflicts, not arbitrary choices).

## Running the pipeline

```bash
# 1. Audio
python3 train_audio_wav2vec2.py \
    --ravdess_dir ravdess/ --crema_d_dir CREMA-D/AudioWAV/ \
    --tess_dir tess_unzipped/ --savee_dir savee_unzipped/AudioData/ \
    --out audio_wav2vec2_model/
python3 predict_audio.py \
    --model_dir audio_wav2vec2_model/ \
    --ravdess_dir ravdess/ --crema_d_dir CREMA-D/AudioWAV/ \
    --tess_dir tess_unzipped/ --savee_dir savee_unzipped/AudioData/ \
    --out audio_probs.npz

# 2. Text
python3 train_text.py --out text_model/
python3 predict_text.py --model_dir text_model/ --out text_probs.npz

# 3. Visual (switch to venv_visual)
python3 build_visual_dataset.py \
    --rafdb_dir archive/DATASET --affectnet_dir "archive (1)/archive (3)" \
    --expw_dir expw/origin --expw_label_lst expw/label.lst \
    --fer2013_dir fer2013_unzipped --out_dir visual_dataset
python3 train_visual_resnet50.py --data_dir visual_dataset --out visual_resnet50_model.h5
python3 predict_visual.py --model_path visual_resnet50_model.h5 --data_dir visual_dataset --out visual_probs.npz

# 4. Fusion evaluation (either venv)
python3 evaluate_fusion.py \
    --audio_probs audio_probs.npz --text_probs text_probs.npz --visual_probs visual_probs.npz \
    --out results/ --n_seeds 20

# 5. Figures for the paper
python3 generate_figures.py \
    --audio_probs audio_probs.npz --text_probs text_probs.npz --visual_probs visual_probs.npz \
    --table5_csv results/table5_corrected.csv --out_dir figures/
```

```
.

└── src/

    ├── audio_label_utils.py          # per-corpus filename -> emotion label parsing
    │

    ├── align_faces_mtcnn.py          # optional MTCNN face re-alignment (see note below)
    │
    ├── train_audio.py                # from-scratch CNN on hand-crafted features (superseded)
    ├── train_audio_wav2vec2.py       # wav2vec2 fine-tuning (used in the paper)
    ├── train_text.py                 # BERT fine-tuning (used in the paper)
    ├── train_text_roberta.py         # RoBERTa fine-tuning (comparison; no gain found)
    ├── train_visual.py               # from-scratch CNN (superseded)
    ├── train_visual_resnet50.py      # ResNet50 transfer learning (used in the paper)
    ├── train_visual_facenet.py       # VGGFace2 transfer learning (negative result; not used)
    │
    ├── predict_audio.py              # inference -> audio_probs.npz
    ├── predict_text.py               # inference -> text_probs.npz
    ├── predict_visual.py             # inference -> visual_probs.npz
    ├── evaluate_fusion.py            # fusion, ablation, missing-modality test
    ├── generate_figures.py           # confusion matrices + results chart
    │
    ├── test_leaky_hypothesis.py      # DIAGNOSTIC ONLY -- reproduces the audio leakage bug
    └── test_leaky_visual_hypothesis.py  # DIAGNOSTIC ONLY -- tests the same for visual
```

The two `test_leaky_*` scripts are deliberately included even though they
demonstrate a *bug*, not a feature — they're what we used to quantify the
leakage effect rather than just assert it mattered, and we think that's
useful for anyone building a similar pipeline to be aware of. **Do not use
their output as a real evaluation result** — they say so loudly in their
own docstrings and console output.

`train_audio.py`, `train_visual.py`, and `train_visual_facenet.py` are kept
for transparency (they document the from-scratch and negative-result paths
that led to the final architecture choices) but are not the models used in
the paper's reported results — see `train_audio_wav2vec2.py` and
`train_visual_resnet50.py` for those.

## Known limitations not yet addressed

- No confidence/reliability weighting in the fusion rule — all modalities
  are weighted equally despite the visual branch being markedly less
  reliable on some classes (see `results/ablation.csv` after running
  `predict_visual.py`).
- The T5-based disgust-class text augmentation has no independent
  verification step confirming paraphrases preserved the original label
  (beyond duplicate/low-diversity filtering).
- `train_visual_resnet50.py`'s exact filter counts for the *original*
  from-scratch CNN baseline (`train_visual.py`) are approximations — the
  source material this was reconstructed from only described the
  architecture qualitatively.
